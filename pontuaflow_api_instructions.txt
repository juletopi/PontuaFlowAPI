OBJETIVO DO SISTEMA (PontuaFlow API)
Implementar uma API simples (sem autenticação) para:
- Projetos
- Weeks (semanas) do projeto (criadas manualmente; numero_semana inserido manualmente)
- Devs dentro de um projeto
- Tasks dentro de um projeto, atribuídas a um Dev e a uma Week
- Listagem de métricas por Dev no projeto:
  - total de pontos
  - aproveitamento (%)
- Ranking dos devs do projeto por total de pontos

REGRAS DE ESCOPO (NÃO IMPLEMENTAR)
- Não implementar autenticação/authorization.
- Não implementar filtros na listagem de tasks/devs e relatórios de exportação de PDF/Execel.
- Não implementar usuários do sistema.

REGRAS E CÁLCULOS (OBRIGATÓRIOS)
1) Pontuação:
- Validar Pontuacao em Task: 0 <= Pontuacao <= 8.
- Valores são 0 = Não deu, 2 = Quase, 3 = Saiu algo, 5 = Deu bom, 8 = Extra.

2) Ranking:
- Ranking de um Project é a lista de Devs do projeto numerados por:
  - total_pontos DESC

3) Total de pontos (por Dev em um Project):
- total_pontos = soma(Task.Pontuacao) das tasks do dev no projeto.

4) Aproveitamento (%):
- min_sucesso = 5
- qtd_tasks = quantidade de tasks do dev no projeto
- pontos_obtidos = soma(Task.Pontuacao) do dev no projeto
- base_sucesso = qtd_tasks * 5
- se qtd_tasks == 0 => aproveitamento = 0
- senão => aproveitamento_percent = (pontos_obtidos / base_sucesso) * 100
- pode ser > 100% (extras)

REQUISITOS DE ENDPOINTS (MÍNIMO VIÁVEL)
Implementar Controllers REST básicos:

A) ProjectsController
- GET /api/projects
- GET /api/projects/{id}
- POST /api/projects
- PUT /api/projects/{id}
- DELETE /api/projects/{id}

B) WeeksController (sempre no contexto do projeto)
- GET /api/projects/{projectId}/weeks
- POST /api/projects/{projectId}/weeks
- PUT /api/projects/{projectId}/weeks/{weekId}
- DELETE /api/projects/{projectId}/weeks/{weekId}

C) DevsController (sempre no contexto do projeto)
- GET /api/projects/{projectId}/devs
- POST /api/projects/{projectId}/devs
- PUT /api/projects/{projectId}/devs/{devId}
- DELETE /api/projects/{projectId}/devs/{devId}

D) TasksController (sempre no contexto do projeto)
- GET /api/projects/{projectId}/tasks
- POST /api/projects/{projectId}/tasks
- PUT /api/projects/{projectId}/tasks/{taskId}
- DELETE /api/projects/{projectId}/tasks/{taskId}

E) MetricsController (leitura)
- GET /api/projects/{projectId}/metrics/devs
  Retorna: para cada dev do projeto:
    - devId, nome, cargo
    - totalPontos
    - aproveitamentoPercent
- GET /api/projects/{projectId}/ranking
  Retorna: lista ordenada por totalPontos DESC (pode reutilizar o endpoint de metrics com ordenação)

F) DevTeamsController
- GET /api/projects/{projectId}/teams
- POST /api/projects/{projectId}/teams
- PUT /api/projects/{projectId}/teams/{teamId}
- DELETE /api/projects/{projectId}/teams/{teamId}
- POST/DELETE para membros e assignments
